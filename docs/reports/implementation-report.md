<a id="phase-1"></a>
## Phase 1: 3Dビューアの基本操作

**目的:** 3Dビューアとしての土台(実解剖メッシュ表示・カメラ操作・断面表示・全体レイアウト)を作る。

**実装内容:**
- 実解剖3Dモデル(`public/models/heart-realistic.glb`)の読み込み。BodyParts3D/Anatomography(ライフサイエンス統合データベースセンター提供、CC BY-SA 2.1 Japan)の実測解剖データから、心臓壁(`HEART`)・右冠動脈(`RCA`)・左前下行枝(`LAD`)・左回旋枝(`LCX`)のメッシュを抽出・結合した単一GLBを使用。
- カメラ操作(ドラッグで回転・パン、ホイールでズーム、視点リセットボタン)。
- 断面表示: X/Y/Z軸のクリッピング平面をスライダーで移動。
- レイアウト: 左サイドパネル + 中央3Dビューアという基本構成。
- モデル読み込みは差し替え可能な設計にしてあり、将来DICOM由来の実メッシュに切り替える余地を残した。

<a id="phase-2"></a>
## Phase 2: 表示コントロール(色・不透明度・セグメント分割)

**目的:** 心臓・血管それぞれの見た目を個別に調整できるようにする。

**実装内容:**
- 心臓の不透明度スライダー、血管(RCA/LAD/LCX)ごとの色ピッカー・不透明度スライダー。
- 「色・不透明度をリセット」ボタン。
- セグメントモード(β): 各血管幹を長さ方向に3分割(近位/中間/遠位)し、区間ごとに個別の色・不透明度を設定できるオプション。ホバーでセグメント名をツールチップ表示。
  - **注意:** 区切り位置は幹の長さに沿った機械的な等分割であり、実際のAHA分類(#1〜#15など)の解剖学的ランドマークとは一致しない(README・コード双方に明記済み)。

**この段階で見つかり修正したバグ:**
- GLTFLoaderが生成するマテリアルは、`transparent`/`opacity`をpropsで変更しただけでは再描画時に反映されない(`material.needsUpdate = true`を明示的に呼ぶ必要があった)。セグメントモードのメッシュでも同様の問題があり、同じ対処をした。

<a id="phase-3"></a>
## Phase 3: シネビュー(X線風投影・拍動ループ・書き出し)

**目的:** 実際の血管造影(アンギオ)のシネ画像に近い、X線透視風の平行投影ビューを追加する。

**実装内容:**
- メインビューとは独立したCanvas(別のWebGLコンテキスト)で、メインビューのカメラ方向を追従する簡易X線風投影(非照明・乗算ブレンドのシルエット表示)を描画。心臓の輪郭をうっすら表示するオプションも追加。
- 拍動アニメーション: 収縮期が速く拡張期が遅い非対称な心周期波形で、メインビュー・シネビュー共通のラッパー(`HeartbeatGroup`)から適用。再生/一時停止に対応。
- GIF(`gifenc`)・PNG連番(`fflate`によるzip)の書き出し機能。
- シネパネルの幅をドラッグで調整できるリサイズハンドル。

**この段階で見つかり修正したバグ:**
- リサイズハンドルがメインビューのCanvasと重なり、ヒットテストが壊れていた。
- drei の `<OrthographicCamera>` がリサイズ時にフラスタムを既定値にリセットしてしまう問題を、カメラを手動管理する方式に置き換えて解消。

<a id="phase-4"></a>
## Phase 4: Cアーム角度システムとリアルな血管撮影風シネモード

**目的:** カテーテル治療(PCI)で実際に使われる「Cアーム角度」(RAO/LAO、CRA/CAUD)をカメラ視点と相互変換できるようにし、より実写に近いシネ表示を追加する。

**実装内容:**
- `utils/cArmAngles.ts`: メインカメラの視線方向とCアーム角度(第一角=RAO/LAO、第二角=CRA/CAUD)を相互変換。DICOM標準に準拠した患者座標系(LPS: X=Right→Left, Y=Anterior→Posterior, Z=Feet→Head)を採用し、vitestで14ケース(往復変換・基準角度)をカバー。
- キャリブレーションは「頭側の軸」+「現在の視点をAP正面として記録」の2アクションで完結する設計(頭側軸プリセットボタン+「この視点をAP正面として設定」ボタン)。AP軸を優先してGram-Schmidt直交化することで、頭側軸の精度が粗くてもAP基準ボタンさえ押せば必ずAP=0°/0°になる。
- サイドパネル常設のCアーム角度表示・スライダー・数値入力・プリセットボタン(`CArmReadout.tsx`)、キャリブレーションUI(`CArmCalibration.tsx`)。
- カメラ角度変更時の球面補間アニメーション、メインビューに軸ギズモを追加。
- リアルな見た目のシネモード: コントラストカーブ・ビネット・フィルムグレインをCanvas2Dで後処理する`utils/cineRealism.ts`(シェーダーを使わないため、ライブ表示・書き出しフレームの両方に同じロジックを適用できる)。

**この段階で見つかり修正したバグ:**
- ストアの初期カメラクォータニオンが、`CameraRig`がマウント時に実際に設定する値と食い違っており、GLB読み込み中の1〜2秒だけ角度表示がずれて見えていた。
- `THREE.Object3D.lookAt()`は`Camera`以外のオブジェクトに対しては向きの計算が反転する仕様があり、初期角度が180度ずれる原因になっていた。

**(参考)このセッションでの検証:** RAO/LAO・CRA/CAUDの符号が逆ではないかという疑いに対し、実際のRCA/LAD/LCXの解剖学的位置(RCAは患者右側、LCXは患者左側という既知の事実)と現在のコードの計算結果を突き合わせて検証し、符号は正しいことを確認済み(反転は見つからなかった)。

<a id="phase-6"></a>
## Phase 6: 病変・治療デバイス配置システム

**目的:** 血管上に狭窄(stenosis)・石灰化(calcification)・ステント(stent)を配置し、リアルな3D形状・シネX線表示で確認できるようにする。

**実装内容(複数コミットにわたる大きな機能追加):**

- **中心線のグラフ化**: 血管中心線を、単一パスではなく「ノード・エッジ・枝(branch)」からなるグラフとして扱うよう再設計。中心線データ自体もオフラインのPythonパイプライン(ボクセル化→skimageによる骨格化→グラフベースの本幹抽出)に切り替え、RCAのように3D空間で複雑に湾曲する血管でも安定して抽出できるようにした(以前のランタイムY軸ビニング方式では破綻していた)。
- **クリック配置UI**: 位置(%)スライダーではなく、3Dビュー上の中心線ノードを直接クリックして病変・デバイスを配置する方式に変更。分岐点をまたぐ移動時は枝の選択肢を提示する。
- **オブジェクト命名の整理**: 元々「lesion(病変)」という名称だったが、ステント(治療デバイス、病変ではない)や将来のガイドワイヤー・カテーテル等のスコープ拡大に伴い、型・ストア・コンポーネント・CSSまで含めて`CardioObject`/`ObjectPanel`/`ObjectMeshes`等へ全面リネーム。サイドパネルも折りたたみ式セクション(`CollapsibleSection`)に再設計。
- **狭窄のジオメトリ**: 血管メッシュ自体を変形する方式(非軸整列血管で歪む問題があった)をやめ、外径チューブ+内径(狭窄後内腔)チューブの中空プラーク管として独立生成する方式に変更。
- **石灰化のジオメトリ**: 当初の「浮遊する三角形の塊」から、血管に沿った部分円筒シェル(外径アーク+内径アーク)へ変更。角度(円周方向の広がり)・向き(心筋方向を基準)を明示的なパラメータとして持たせた。
- **シネX線モードの本格実装**: 深度ピール(前面/背面の深度差から光学的厚みを求める手法)によるポストプロセスパイプラインを新設。血管・心臓・石灰化・ステントそれぞれの吸収係数を個別設定可能にし、Beer-Lambert則で合成。
- **ステントの網目構造**: 当初の無地の円筒から、ダイヤモンドカット状のストラット網目ジオメトリへ変更(このセッションでさらにUltimaster準拠のオープンセル・リング構造へ作り直し。詳細は[Phase 7以降の継続的な品質改善](#post-phase-7)参照)。

**この段階で見つかり修正した主なバグ:**
- シネX線が完全に真っ黒になる不具合: フラグメントシェーダーが同時に21個のテクスチャサンプラーを要求しており、GPU/ドライバのMAX_TEXTURE_IMAGE_UNITS(16)を超過してWebGLプログラムのリンクに失敗していた。血管・心臓・オブジェクトそれぞれ「1グループにつき共有アキュムレータ1枚」に加算ブレンドで積算する方式に変更し、オブジェクト数が増えてもテクスチャ数が増えない構造にした。
- フル32bit float型の深度ピールターゲットが一部のWindows/ANGLE環境で描画されない問題(HalfFloatTypeに変更して解消)。
- 血管ごとの「近位度」テーパー処理が、視線方向に沿って走る血管で前面/背面の深度ピール交差を非対称にスケールし、急角度(CRA85など)でほぼ消失する不具合。テーパー処理自体を廃止して解消。
- 石灰化シェルの長さ方向テーパーが、狭窄の実装を安易に流用したために誤って両端に適用され、円錐状に先細る不具合。
- 石灰化表面の凹凸ノイズが、深度ピールと組み合わさることで多数の小さな黒い斑点に分裂して見える不具合。ノイズ自体を廃止して解消。

<a id="phase-7"></a>
## Phase 7: 造影剤フローシミュレーション

**目的:** 「造影剤を注入」操作で、ボーラス(造影剤の塊)が起始部から末梢へ向かって血管グラフに沿って広がっていく様子を、臨床的に正しい血流制限モデルとともに再現する。

**実装内容:**
- 起始部(各血管本幹のt=0)からボーラスが発生し、中心線グラフの枝を親→子の順(BFS)にたどりながら伝播。側枝は親枝がその分岐点に到達した時刻から自分の到達時刻計算を開始する。
- 狭窄・石灰化の血流への影響を、教科書的な冠循環生理に沿って2つの側面でモデル化:
  - 伝播速度は局所内腔断面積比率に応じて低下する(狭窄区間の通過に時間がかかる)。
  - 「通過係数」という概念を新設: 狭窄より末梢に到達しうる濃度の上限を、%狭窄率に対するロジスティック曲線(軽度〜中等度では平坦、70%を超えると急激に低下、完全閉塞で厳密に0)で頭打ちにする。これは「冠血流予備能は高度狭窄になるまで保たれる」という臨床知見に対応する。
- スキーマ表示・メイン3Dビュー・シネX線の3つの表示すべてで、同じ伝播計算結果を共有して同期表示。

**この段階で見つかり修正した主なバグ:**
- `buildTubeFromPoints`の既定の強い平滑化(24回)が、ステント等の短い局所区間向けにチューニングされたものであったにもかかわらず、実際の血管の湾曲全体にまたがる枝にそのまま適用され、造影剤チューブが実際の中心線からずれて描画されていた。
- メインビューの造影剤チューブが血管本来の色をそのまま使っており、「造影されている区間」と「されていない区間」が区別できなかった(濃度に応じた頂点カラーを焼き込む方式に変更して解消)。
- 濃度マスクがグローバル(3血管共通)に適用されており、画面上である血管が別の未造影血管と重なった際に、未造影側が造影側の高い濃度に「便乗」して写ってしまう不具合。血管ごとに個別マスクを適用する方式に変更して解消。
- 狭窄部で伝播フロントが視覚的に停止して丸いブロブとして溜まって見える不具合(伝播モデルを「フロント速度を落とす」から「到達しうる最大濃度を下げる」方式に転換して解消)。
- 3Dビュー・シネX線双方で、チューブの半径/厚みを生の濃度でスケールしていたため、狭窄自身の位置(その区間はceilingが既に下がっている)で二重に絞られ、くびれがほぼ消失していた。局所濃度の「その上限に対する立ち上がり具合」で幅を決め、薄さ自体は別経路(頂点カラー、または血管自身の濃度マスク)で表現するよう修正。

<a id="post-phase-7"></a>
## Phase 7以降の継続的な品質改善

Phase 7の後も、臨床的な正確性・見た目のリアリズムを高めるための修正を継続している(明示的な新Phase番号は付けず、既存Phaseの精度向上として実施)。

- **RCAオスティウム(起始部)の判定修正**: 根(起始部)を「ワールドY座標最大の端点」で選んでいたため、RCAでは円錐枝/洞結節枝相当の細い枝先が誤って選ばれることがあった。「内腔半径が最大の端点」を根とする方式に変更(オスティウムは解剖学的にその血管で最も内腔径が太い点であるため)。
- **石灰化のシネX線描画の修正**(3ラウンドにわたるユーザーフィードバックを反映):
  - 石灰化シェルの外径面・内径面が同一の三角形の巻き順を共有しており、深度ピールの前面/背面パスがどちらも「背面」として分類され、実際の材質厚みではなく視点からの距離スケールの値になっていた(石灰化がどれだけ吸収係数を下げても真っ黒に写り、端に輪郭が残る不具合の根本原因)。外径面の巻き順を反転して解消。
  - 石灰化シェルに長さ方向のテーパーがなく、区間境界にリング状の輪郭が残っていた。狭窄と同じ台形テーパー(`lesionTaperProfile`)を適用して解消。
  - 石灰化の吸収係数を、実際の透視での見え方(造影された血管より明確に淡い陰影)に合わせて複数回チューニング(35→70→8→3)。
- **造影剤フローの回帰テスト修正**: 孫枝(側枝の側枝)の分岐判定が、直接の親枝上の位置を本幹上の位置と取り違えていた。祖先を本幹までたどって正しい本幹上の分岐位置を求めるよう修正。
- **ステントの網目構造の刷新**: 実在の薬剤溶出ステント「Terumo Ultimaster」(コバルトクロム、ストラット厚80µm、オープンセル・2リンク設計)の構造を調査した上で、従来の「編み込みワイヤーメッシュ」型ジオメトリを、独立した正弦波状リング+少数のコネクタで接続するオープンセル構造に全面的に作り直した。

<a id="caveats"></a>
## 既知の制約・注意点

- **セグメントモードの区切り**: 血管を機械的に3等分しているだけで、実際のAHA分類(#1〜#15等)の解剖学的ランドマークとは一致しない(Phase 2)。
- **心臓の動き**: 拍動アニメーションは実測モーションデータではなく、非対称な数式ベースの近似(Phase 3)。
- **モデルの出典**: `heart-realistic.glb`はBodyParts3D由来のCC BY-SA 2.1 Japanライセンスであり、利用時は指定の表記が必要(README参照)。
- **Phase番号の欠番**: 前述の通り「Phase 5」は履歴上存在しない。

<a id="future"></a>
## 未着手の項目(Phase 8 以降)

コード中のコメントで、次のPhaseとして以下が想定されている(未実装):

- **Phase 8: 心筋灌流(myocardial perfusion)** — 「どの枝がどの心筋領域を灌流するか」「狭窄により虚血になっているか」を判定する機能。Phase 7で構築した枝ごとの到達時刻テーブル・血流制限(ceiling)の仕組みをそのまま再利用できるよう、`utils/contrastFlow.ts`側で既に想定した設計になっている(`getMaxStenosisSeverity`等)。
