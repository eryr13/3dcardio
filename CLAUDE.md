# このプロジェクトについて

3D心臓・冠動脈ビューア(React Three Fiber)。冠動脈やPCIデバイス(ガイディングカテーテル・ガイドワイヤー等)を、実際の心臓メッシュ(ImageCAS由来のheart-realistic.glb、public/models/)や中心線データ(src/data/centerlines.json)を基に手続き的に配置・生成している。大動脈本体のメッシュは元データに含まれていないため、大動脈基部・上行大動脈・弓部・下行大動脈は冠動脈入口部の実座標から幾何学的に逆算した補助表示(src/components/models/aorticRootMesh.ts)。

## 3Dビューに関わる変更をしたら、必ず自分の目で見て確認する

このプロジェクトでは、コードやテストが通っていても、実際にレンダリングすると物理的・解剖学的に不自然な見た目になっていることがある(例: デバイスが心筋メッシュを貫通している、血管と大動脈の位置関係がずれている、経路が破綻して見える等)。`tsc`/`vitest`/`oxlint`が通ることは前提条件であって、完了の証明にはならない。

3Dビュー上の位置関係(デバイス・血管・心臓メッシュ・大動脈などの幾何学的配置)に影響する変更をしたら、**「完了」と報告する前に、必ず以下を行うこと**:

1. 開発サーバーを起動する(`npx vite --port 5183` 等。既に起動していないか先に確認する)。
2. Playwrightでアプリを操作し、変更が関わる機能を実際に画面上で再現する(該当するトグルをON、対象血管を切り替える、挿入progressを最大にする等)。心臓メッシュを非表示/低不透明度にする、カメラをズーム・回転するなどして、問題の箇所がはっきり見える構図にする。
3. スクリーンショットを撮り、**Readツールで実際に画像を読んで自分の目で見る**。ログや数値検証(距離・内包率など)は補助的な裏付けであり、見た目の自然さを確認したことにはならない——必ず画像を見ること。
4. 見た目が物理的・解剖学的におかしい場合(貫通、断絶、明らかに不自然な位置・スケール等)は、ユーザーに聞く前に自分で原因を調べて修正し、再度スクリーンショットで確認する。これをおかしくなくなるまで繰り返す。
5. 検証用に作った一時スクリプト(`__verify_*.mjs` 等、リポジトリ直下に置く)は、確認が終わったら必ず削除する。

### この確認の限界(正直に伝えること)

- 「メッシュを貫通している」「明らかに変な位置にある」といった物理的な破綻は目視で気づいて直せることが多い。
- 「このカテーテルの曲がり方は臨床的にJL4として正確か」のような、専門知識が必要な微妙な妥当性判断は目視確認だけでは限界がある。気づいた範囲で直したうえで、確認した内容と限界をユーザーに正直に報告すること。

## 開発時の実務メモ

- 過去のセッションではdevサーバーをポート5183で起動して確認してきた。
- Canvasは常時再描画されるため、Playwrightの`locator.screenshot()`は要素の安定待ちでハングしやすい。`page.screenshot({ clip: box })`(`box = await canvas.boundingBox()`)を使うこと。
- パネルの「大動脈基部」「ガイドワイヤー・カテーテル」等はdetails/summaryで折りたたまれていることがあるので、対象のsummaryをクリックしてから操作する。
